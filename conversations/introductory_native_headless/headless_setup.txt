# Headless Android Development: Environment Check & Test Guide

## Quick Environment Check

Run these commands in terminal/cmd to assess your setup:

```bash
# 1. Check if ANDROID_HOME is set
echo $ANDROID_HOME          # Linux/Mac
echo %ANDROID_HOME%         # Windows CMD
$env:ANDROID_HOME           # PowerShell

# 2. Check if SDK tools are in PATH
adb version
sdkmanager --version
avdmanager list avd

# 3. Check build tools
ls $ANDROID_HOME/build-tools/ 2>/dev/null || dir "%ANDROID_HOME%\build-tools" 2>nul

# 4. Check platforms
ls $ANDROID_HOME/platforms/ 2>/dev/null || dir "%ANDROID_HOME%\platforms" 2>nul
```

## Find SDK if Variables Not Set

Android Studio typically installs SDK in these locations:

| OS | Default Path |
|----|--------------|
| Windows | `%LOCALAPPDATA%\Android\Sdk` |
| Mac | `~/Library/Android/sdk` |
| Linux | `~/Android/Sdk` |

```bash
# Quick search on Windows (run in project root or parent)
dir /s /b "%LOCALAPPDATA%\Android\Sdk" 2>nul | findstr /i "adb.exe" | head -1
```

## Practical Test: Build Your Project from Terminal

```bash
# Navigate to your project
cd /path/to/your/android/project

# Option 1: Using Gradle wrapper (recommended - no setup needed)
./gradlew assembleDebug          # Linux/Mac
gradlew.bat assembleDebug        # Windows

# Option 2: If wrapper doesn't exist, use system Gradle (rarely needed)
gradle assembleDebug

# Check output
ls app/build/outputs/apk/debug/
```

## Test Run on Emulator (Headless-ish)

```bash
# 1. List available AVDs
avdmanager list avd

# 2. Start emulator in headless mode
emulator -avd <AVD_NAME> -no-window -no-audio -gpu off

# Alternative: start without GPU and with low resources
emulator -avd <AVD_NAME> -no-window -no-audio -no-snapshot -memory 1024

# 3. Wait for boot (check with adb)
adb wait-for-device
adb shell getprop sys.boot_completed  # Returns "1" when ready

# 4. Install and run
adb install app/build/outputs/apk/debug/app-debug.apk
adb shell am start -n com.your.package/.MainActivity
```

## Test Run on Physical Device via ADB (Truly Headless)

```bash
# Via USB (enable USB debugging on device first)
adb devices
adb install app/build/outputs/apk/debug/app-debug.apk

# Via WiFi (after initial USB connection or if already paired)
adb tcpip 5555
adb connect <DEVICE_IP>:5555
adb install app/build/outputs/apk/debug/app-debug.apk
```

## For Coding Agents: Essential Scripts

Create `build_and_run.sh` (or `.bat` for Windows):

```bash
#!/bin/bash
set -e

PROJECT_DIR="$1"
PACKAGE_NAME="$2"  # e.g., com.example.myapp
ACTIVITY_NAME="$3" # e.g., .MainActivity

echo "=== Building ==="
cd "$PROJECT_DIR"
./gradlew assembleDebug

APK_PATH="app/build/outputs/apk/debug/app-debug.apk"
echo "=== Installing ==="
adb install -r "$APK_PATH"

echo "=== Launching ==="
adb shell am start -n "$PACKAGE_NAME/$ACTIVITY_NAME"

echo "=== Done ==="
```

## What to Look For in Output

| Check | ✅ Good | ❌ Problem |
|-------|---------|------------|
| `ANDROID_HOME` | Set to valid path | Empty or wrong path |
| `adb` | Found in PATH | "NOT in PATH" |
| `build-tools/` | Has version folder (e.g., `34.0.0`) | Empty or missing |
| `platforms/` | Has `android-XX` folder | Empty or missing |
| AVD list | Shows device names | "None found" |
| `adb devices` | Shows device ID | "no devices" |

# Fix: There's a Typo in Your Command

You're missing the dash on `-n`:

```bash
# ❌ Your command (missing dash)
adb shell am start n com.example.taskmanager/.MainActivity

# ✅ Correct command
adb shell am start -n com.example.taskmanager/.MainActivity
```

That's why it says `pkg=n` — it parsed "n" as the package name.

## Quick Fix: Set ANDROID_HOME

Since you found adb, add this to your shell config:

```bash
# For zsh (default on modern Mac)
echo 'export ANDROID_HOME="$HOME/Library/Android/sdk"' >> ~/.zshrc
echo 'export PATH="$PATH:$ANDROID_HOME/platform-tools"' >> ~/.zshrc
source ~/.zshrc
```

## Missing sdkmanager / avdmanager

Those live in `cmdline-tools`, which is a separate install from `platform-tools`. You likely don't have it.

**Check if it exists anywhere:**
```bash
find ~/Library/Android/sdk -name "sdkmanager" 2>/dev/null
```

**If not found**, you have two options:

### Option 1: Install cmdline-tools (for emulator setup)

```bash
# Download from Google
cd /tmp
curl -o cmdline-tools.zip "https://dl.google.com/android/repository/commandlinetools-mac-11076708_latest.zip"

# Unzip and move to correct location
unzip cmdline-tools.zip
mkdir -p ~/Library/Android/sdk/cmdline-tools
mv cmdline-tools ~/Library/Android/sdk/cmdline-tools/latest

# Add to PATH
echo 'export PATH="$PATH:$ANDROID_HOME/cmdline-tools/latest/bin"' >> ~/.zshrc
source ~/.zshrc

# Verify
sdkmanager --version
```

### Option 2: Skip it entirely

Since you can already:
- ✅ Build with `./gradlew`
- ✅ Install via `adb` to physical device

You **don't need** sdkmanager/avdmanager unless you want to create/manage emulators from terminal. For a coding agent with a physical device, you're already set.

```bash

# 1. Build

# 5. Check logs if something goes wrong
adb logcat -d | grep -i "taskmanager" | tail -20
```

## If You Still Want Emulator Later

Once cmdline-tools are installed:
```bash
# Accept licenses
yes | sdkmanager --licenses

# Install a system image (pick one)
sdkmanager "platforms;android-34" "system-images;android-34;google_apis;x86_64"

# Create AVD
avdmanager create avd -n test_device -k "system-images;android-34;google_apis;x86_64" -d "pixel_6"

# Run headless
emulator -avd test_device -no-window -no-audio -gpu swiftshader_indirect
```

But for now, fix the `-n` typo and you should be good to go with your physical device.
